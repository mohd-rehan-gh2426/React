# JavaScript Comparison Operators

Comparison operators are used to **compare two values** in JavaScript. They are primarily used with numbers, but can also be used with strings. However, comparing strings directly with numbers is **not recommended**; instead, convert the string to a number first.

Comparison operators always return **Boolean values**: `true` or `false`.

---

## 1. Equality Operator (`==`)

* Checks if two values are equal.
* Performs **implicit type conversion** (automatic conversion of types if they differ).

```javascript
const user1Age = 18; // number
const user2Age = "18"; // string

console.log(user1Age == user2Age); // true (string "18" is converted to number 18)
```

### Key Notes:

* `==` is called **normal equality**.
* It converts one value to match the type of the other before comparison (implicit/automatic conversion).
* Can lead to **unexpected behavior**, especially with strings and numbers.

---

## 2. Strict Equality (`===`)

* Checks if both **value and type** are equal.
* **No automatic conversion** occurs. You must convert manually if types differ.

```javascript
console.log(user1Age === user2Age); // false (number !== string)
```

### Explicit Conversion Examples:

```javascript
// Convert string to number manually
console.log(user1Age === Number(user2Age)); // true
console.log(user1Age === +user2Age);       // true using unary plus
```

✅ **Best practice:** Always use **strict equality (`===`)** to avoid unpredictable behavior.

---

## 3. Not Equal (`!=`) and Strict Not Equal (`!==`)

* `!=` → Checks if values are **not equal** (implicit conversion applied).
* `!==` → Checks if **value or type is not equal** (strict comparison, no conversion).

```javascript
console.log(18 != 24);      // true
console.log(18 !== "18");   // true (different types)
```

---

## 4. Greater Than (`>`) and Less Than (`<`)

* Used to compare numbers.
* JavaScript performs **automatic conversion** if a value is a string containing a number.

```javascript
const user1 = 21;
const user2 = 24;

console.log(user2 > user1); // true
console.log(user1 > user2); // false
```

### Notes:

* There is **no strict version** of `>` or `<`.
* Always convert strings to numbers manually if comparing strings with numbers:

```javascript
console.log(Number("24") > 21); // true
```

---

## 5. Greater Than or Equal To (`>=`) and Less Than or Equal To (`<=`)

* Checks if a value is greater/less than **or equal to** another value.

```javascript
console.log(user1 >= 21); // true
console.log(user1 <= 24); // true
```

* Works with numbers and strings that can be converted to numbers (automatic conversion happens for strings).
* No strict version exists for these operators, so manual conversion is safer.

---

## 6. Implicit vs Explicit Conversion

* **Implicit Conversion:** JavaScript automatically converts types behind the scenes.

  * Example: `"18" == 18` → `"18"` converted to number 18.
* **Explicit Conversion:** You manually convert the type before comparison.

  * Example: `Number("18") === 18`

✅ **Rule of thumb:** Always do explicit conversion to avoid unpredictable results.

---

## 7. Summary of Comparison Operators

| Operator | Description           | Implicit Conversion? | Example               |
| -------- | --------------------- | -------------------- | --------------------- |
| `==`     | Equal                 | Yes                  | `18 == "18"` → true   |
| `===`    | Strict Equal          | No                   | `18 === "18"` → false |
| `!=`     | Not Equal             | Yes                  | `18 != "24"` → true   |
| `!==`    | Strict Not Equal      | No                   | `18 !== "18"` → true  |
| `>`      | Greater Than          | Yes                  | `24 > "21"` → true    |
| `<`      | Less Than             | Yes                  | `21 < 24` → true      |
| `>=`     | Greater Than or Equal | Yes                  | `24 >= "24"` → true   |
| `<=`     | Less Than or Equal    | Yes                  | `21 <= 24` → true     |

---

## 8. Best Practices

1. **Always prefer strict equality (`===`)** over normal equality (`==`) to avoid unexpected behavior.
2. **Explicitly convert types** if needed:

   * Use `Number()` or unary `+` to convert strings to numbers.
   * Use `String()` to convert numbers to strings if required.
3. Avoid relying on **automatic type conversion**, it can be unpredictable.
4. Comparison operators always return a **Boolean** (`true` or `false`).

---

## 9. Example Code

```javascript
const user1Age = "24"; // string
const user2Age = 24;   // number

// Normal equality (implicit conversion)
console.log(user1Age == user2Age); // true

// Strict equality (no conversion)
console.log(user1Age === user2Age); // false

// Manual conversion
console.log(Number(user1Age) === user2Age); // true

// Comparison
console.log(user2Age > user1Age); // true (string converted to number)
console.log(user1Age < user2Age); // true
```

## **1. Logical Operators Overview**

In JavaScript, **logical operators** are used to combine or invert boolean values (true/false). There are **three main types**:

| Operator | Symbol | Description                                            |   |                                          |
| -------- | ------ | ------------------------------------------------------ | - | ---------------------------------------- |
| AND      | `&&`   | True **only if both operands are true**                |   |                                          |
| OR       | `      |                                                        | ` | True if **at least one operand is true** |
| NOT      | `!`    | Inverts the boolean value (true → false, false → true) |   |                                          |

---

## **2. AND (`&&`) Operator**

* Checks **two values**.
* Returns **true** if **both** are true.
* Returns **false** if **at least one** is false.

### **Basic Example**

```js
console.log(true && true);   // true
console.log(true && false);  // false
console.log(false && true);  // false
console.log(false && false); // false
```

### **Using comparison operators**

```js
console.log(6 > 5 && 8 > 4); // true, both conditions are true
console.log(6 > 5 && 8 < 4); // false, second condition is false
```

### **Practical Example: Age Check**

Check if a user's age is between 18 and 24:

```js
const userAge = 25;
const isCollegeStudent = userAge >= 18 && userAge <= 24;
console.log(isCollegeStudent); // false, 25 is outside the range

const userAge2 = 22;
const isCollegeStudent2 = userAge2 >= 18 && userAge2 <= 24;
console.log(isCollegeStudent2); // true
```

**Key takeaway:**
AND requires **both sides** to be true for the result to be true.

---

## **3. OR (`||`) Operator**

* Checks **two values**.
* Returns **true** if **at least one** is true.
* Returns **false** only if **both** are false.

### **Basic Example**

```js
console.log(true || true);   // true
console.log(true || false);  // true
console.log(false || true);  // true
console.log(false || false); // false
```

### **Practical Example: Check Student**

```js
const isSchoolStudent = userAge >= 5 && userAge <= 18;
const isCollegeStudent = userAge >= 18 && userAge <= 24;

const isStudent = isSchoolStudent || isCollegeStudent;
console.log(isStudent); // true if user is either school or college student
```

**Key takeaway:**
OR requires **only one side** to be true for the result to be true.

---

## **4. Truthy and Falsy Values**

In JavaScript, logical operators don’t always return `true` or `false`—they return the **actual value** of operands.
Values are considered **false** if they are:

```
0, null, undefined, NaN, "", false
```

Everything else is **truthy**.

### **Example**

```js
console.log(0 && 5);      // 0 → AND stops at first falsy
console.log(0 || 5);      // 5 → OR finds the first truthy
console.log("" && "hello"); // "" → AND stops at falsy
console.log("" || "hello"); // "hello" → OR finds truthy
```

---

## **5. How AND and OR Evaluate Expressions**

* **AND (`&&`)**: stops at the **first falsy** value and returns it. If all are truthy, returns the last value.
* **OR (`||`)**: stops at the **first truthy** value and returns it. If all are falsy, returns the last value.

```js
console.log(2 && 3); // 3 → both truthy, returns last
console.log(0 && 3); // 0 → first falsy
console.log(0 || 3); // 3 → first truthy
console.log(null || undefined); // undefined → both falsy, returns last
```

---

## **6. NOT (`!`) Operator**

* Reverses the boolean value.
* True → False, False → True.

```js
console.log(!true);  // false
console.log(!false); // true

// Works with truthy/falsy values
console.log(!0);    // true, because 0 is falsy
console.log(!"");   // true, empty string is falsy
console.log(!"Hello"); // false, non-empty string is truthy
```

* Double NOT (`!!`) converts **any value to boolean**:

```js
console.log(!!0);      // false
console.log(!!"Hello"); // true
```

---

## **7. Advanced Notes**

* AND (`&&`) can **execute expressions**:

```js
const result = console.log("Hi") && 5; 
// prints "Hi", result is 5 because first part is executed
```

* OR (`||`) can **short-circuit** if first value is truthy:

```js
const result = "Hello" || console.log("Hi");
// "Hello" is returned, second part is not executed
```

* **Bitwise AND (`&`) and OR (`|`)** exist, but rarely used for normal logic—they work on binary representations.

---

## **8. Summary Table**

| Operator | Returns True When         | Short-circuit Behavior |                                |                       |
| -------- | ------------------------- | ---------------------- | ------------------------------ | --------------------- |
| `&&`     | Both operands are truthy  | Stops at first falsy   |                                |                       |
| `        |                           | `                      | At least one operand is truthy | Stops at first truthy |
| `!`      | Always inverses the value | N/A                    |                                |                       |

---

✅ **Key tips to remember:**

1. AND = both must be true.
2. OR = at least one must be true.
3. NOT = flips true/false.
4. JavaScript uses **truthy/falsy values**, not just `true/false`.
5. Logical operators can evaluate any expression, not just booleans.

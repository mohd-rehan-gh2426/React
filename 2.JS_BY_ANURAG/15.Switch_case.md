# JavaScript Switch Case (Complete Notes)

---

## 1. Why `switch case`?

When multiple conditions check the **same variable**, `if-else-if` becomes repetitive and messy.

### Example using if-else:
```js
if (dayNumber === 0) {
  console.log("Sunday");
} else if (dayNumber === 1) {
  console.log("Monday");
} else if (dayNumber === 2) {
  console.log("Tuesday");
} else {
  console.log("Invalid day");
}
````

‚ùå Repetitive
‚ùå Less readable

‚úÖ Solution ‚Üí `switch case`

---

## 2. What is `switch case`?

`switch` is used to:

* Compare **one value**
* Against **multiple fixed values**
* In a **clean and readable way**

Think:

> Match one value ‚Üí execute matching case

---

## 3. Syntax of `switch`

```js
switch (expression) {
  case value1:
    // code
    break;

  case value2:
    // code
    break;

  default:
    // code
}
```

### Important Rules

* Uses **strict equality (`===`)**
* `break` stops execution
* `default` runs if no case matches

---

## 4. Day Number Example

### Variable

```js
const dayNumber = 3;
```

### Switch Code

```js
switch (dayNumber) {
  case 0:
    console.log("Sunday");
    break;

  case 1:
    console.log("Monday");
    break;

  case 2:
    console.log("Tuesday");
    break;

  case 3:
    console.log("Wednesday");
    break;

  case 4:
    console.log("Thursday");
    break;

  case 5:
    console.log("Friday");
    break;

  case 6:
    console.log("Saturday");
    break;

  default:
    console.log("Please enter a valid day number");
}

console.log("Program Ended");
```

---

## 5. Why is `break` Important?

Without `break`, JavaScript executes **all cases below the matched case**.

### Example (No break)

```js
case 1:
  console.log("Monday");

case 2:
  console.log("Tuesday");
```

### Output (if dayNumber = 1)

```
Monday
Tuesday
Wednesday
Thursday
Friday
Saturday
```

üìå This is called **fall-through**

---

## 6. Role of `break`

```js
break;
```

* Exits the switch block
* Prevents fall-through
* Ensures **only one case runs**

üëâ Almost always required

---

## 7. `default` Case

`default` works like `else`.

```js
default:
  console.log("Invalid input");
```

### Notes

* Optional
* Usually written at the end
* `break` not required if last

---

## 8. Order of Cases

Order **does not matter**.

```js
case 6:
case 0:
```

Switch checks for **matching**, not sequence.

---

## 9. How `switch` Works Internally

```js
switch (value)
```

Means:

```js
value === caseValue
```

‚úîÔ∏è Type and value must match

## 10. `switch(true)` ‚Äî Advanced Usage

Used to replace complex `if-else-if` logic.

---

### If-Else Example

```js
if (age < 18) {
  console.log("School student");
} else if (age >= 18 && age <= 24) {
  console.log("College student");
} else if (age >= 25 && age <= 45) {
  console.log("Working professional");
} else {
  console.log("Invalid age");
}
```

---

### Converted to `switch(true)`

```js
switch (true) {
  case age < 18:
    console.log("School student");
    break;

  case age >= 18 && age <= 24:
    console.log("College student");
    break;

  case age >= 25 && age <= 45:
    console.log("Working professional");
    break;

  default:
    console.log("Please enter a valid age");
}
```

### Why it works?

* `switch(true)` looks for the **first true case**
* Behaves like `else-if`

---

## 11. Important Rule in `switch(true)`

* Multiple conditions may be true
* Only the **first true case executes**
* Because of `break`

---

## 13. Strict Comparison (`===`)

Switch uses strict comparison.

```js
1 !== "1"
"A" !== "a"
false !== null
```

Type mismatch ‚Üí no match

---

## 14. Handling Case Sensitivity

### Best Practice

```js
switch (grade.toLowerCase()) {
  case "a":
    console.log("Score between 85% and 100%");
    break;

  case "b":
    console.log("Score between 75% and 85%");
    break;

  case "c":
    console.log("Score between 60% and 75%");
    break;

  default:
    console.log("Failed");
}
```

‚úîÔ∏è Works for `A` or `a`

---

## 15. Switch vs If-Else

### Use `switch` when:

* One variable
* Many fixed values
* Clean readability needed

### Use `if-else` when:

* Conditions are complex
* Logical operators dominate
* Better clarity with ranges

---

## 16. Key Takeaways

* `switch` uses `===`
* `break` prevents fall-through
* `default` = `else`
* `switch(true)` mimics `else-if`
* First matching case executes
* Normalize strings using `toLowerCase()`
* Avoid missing `break`

# JavaScript Nested if-else (Notes)

## 1. Recap: if / else / else-if

JavaScript uses `if`, `else if`, and `else` to make decisions.

```js
if (condition1) {
  // runs if condition1 is true
} else if (condition2) {
  // runs if condition1 is false and condition2 is true
} else {
  // runs if none of the above are true
}
````

### Important Rule

* Only **one block** runs in an `if-else-if` chain.
* Once a condition is true, JavaScript **stops checking** further conditions.

---

## 2. What is Nesting?

**Nesting** means writing an `if-else` **inside another `if` or `else` block**.

```js
if (condition1) {
  if (condition2) {
    // nested if
  } else {
    // nested else
  }
}
```

Think of it as:

> If this is true, then check another condition.

---

## 3. Example Setup (Hardcoded Values)

```js
let username = "Anurag";
let age = 22;
```

Hardcoding values helps in learning and debugging faster.

---

## 4. Age-based if-else-if Structure

```js
if (age < 5) {
  console.log("Too young");
} else if (age >= 5 && age <= 17) {
  console.log("School student");
} else if (age >= 18 && age <= 24) {
  console.log("College student");
} else {
  console.log("Working professional");
}
```

If `age = 22`, JavaScript enters:

```js
else if (age >= 18 && age <= 24)
```

---

## 5. Nested if inside else-if

```js
else if (age >= 18 && age <= 24) {
  console.log("College student");

  if (age > 20) {
    console.log("He is above 19");
  } else {
    console.log("He is below 20");
  }
}
```

### Explanation

* First condition filters age between `18–24`
* Inside that block, another decision is made
* Inner `if-else` works **independently**

---

## 6. Rules of Nested if-else

* If `if` runs, its `else` will **not run**
* If `if` is false, `else` will run
* Same rules apply at every nesting level

---

## 7. Adding More Nesting (Username Length Check)

1. Age category
2. Age comparison (`> 20`)

---

## 8. `username.length`

```js
username.length
```

* Returns number of characters in a string
* `"Anurag"` → `6`
* `"Atul"` → `4`

Used to check string size in conditions.

---

## 9. IMPORTANT: Conditions That Will Never Run

### Wrong Logic Example

```js
else if (age >= 5 && age <= 17) {
  if (age > 20) {
    console.log("This will never run");
  }
}
```

### Why?

* This block runs only when `age` is `5–17`
* `age > 20` can **never** be true here

JavaScript already filtered the value.

---

## 10. Key Concept: Condition Filtering

Once a condition is matched, JavaScript **assumes that condition is true** inside the block.

```js
else if (age >= 18 && age <= 24) {
  // age is guaranteed to be between 18 and 24 here
}
```

Avoid checking impossible conditions inside filtered blocks.

---

## 11. Complete Working Example

```js
let username = "Atul";
let age = 19;

if (age < 5) {
  console.log("Too young");
} else if (age >= 5 && age <= 17) {
  console.log("School student");
} else if (age >= 18 && age <= 24) {
  console.log("College student");

  if (age > 20) {
    console.log("He is above 19");
  } else {
    console.log("He is below 20");

    if (username.length > 5) {
      console.log("Username length is greater than 5");
    } else {
      console.log("Username is very short");
    }
  }
} else {
  console.log("Working professional");
}
```

---

## 12. When to Use Nesting?

### Use Nesting When:

* One condition depends on another
* Step-by-step filtering is needed

### Avoid Deep Nesting When:

* Code becomes hard to read
* `else-if` or `switch` can be used instead

---

## 13. Key Takeaways

* Nested `if-else` means `if` inside another `if` or `else`
* Only one `if-else-if` block runs
* Inner conditions work independently
* JavaScript filters values before entering a block
* Never write impossible conditions inside filtered blocks
* Prefer readability over deep nesting

## 1️⃣ **Decision-Making in JavaScript: if, else if, else**

In JavaScript, we can make our program **choose what to do based on a condition** using `if`, `else if`, and `else`.

### Basic Structure:

```javascript
if (condition1) {
    // Code runs if condition1 is true
} else if (condition2) {
    // Code runs if condition1 is false but condition2 is true
} else {
    // Code runs if none of the above conditions are true
}
```

**Key Points:**

* The first `if` is **mandatory**.
* `else if` is optional, and you can have multiple `else if`.
* `else` is optional and **cannot exist without an `if`**.
* Once a condition is satisfied, **no further `else if` or `else` blocks are checked**.

---

### Example: Age categories

```javascript
let age = 20;

if (age >= 0 && age <= 4) {
    console.log('User is a kid');
} else if (age >= 5 && age <= 17) {
    console.log('User is a teenager');
} else if (age >= 18 && age <= 45) {
    console.log('User is an adult');
} else {
    console.log('User is retired or immortal');
}
```

✅ **How it works:**

* If `age = 3` → prints `User is a kid` → stops checking further.
* If `age = 20` → prints `User is an adult`.
* If `age = 130` → prints `User is retired or immortal`.

## 2️⃣ **Optimization Using `else if`**

Without `else if`:

```javascript
if (age >= 0 && age <= 4) {
    console.log('Kid');
}
if (age >= 5 && age <= 17) {
    console.log('Teenager');
}
```

* Even if the first `if` is true, the program **still checks all other conditions**.
* This wastes processing.
* Using `else if` **stops checking once a condition is true**, optimizing your code.

## 3️⃣ **Logical Operators: OR (`||`) and AND (`&&`)**

### OR (`||`)

* Returns the **first truthy value** or the last value if all are falsy.

```javascript
let name = prompt('Enter your name') || 'PROCODER';
```

* If the user enters nothing → `prompt()` returns `null` → falsy → `name` becomes `'PROCODER'`.

### AND (`&&`)

* Returns the **first falsy value**, or the last value if all are truthy.

```javascript
let isAdult = age >= 18 && 'Yes';
```

* If age ≥ 18 → returns `'Yes'`
* If age < 18 → returns `false` (or the first falsy value)

**Tip:** OR is great for **default values**, AND is great for **checking conditions before executing**.

## 4️⃣ **Truthy and Falsy Values**

In JavaScript, certain values are **false in a Boolean context**:

**Falsy values:**

* `false`
* `0`
* `''` (empty string)
* `null`
* `undefined`
* `NaN`

Everything else is **truthy**.

```javascript
if (12) {
    console.log('Hello'); // runs, because 12 is truthy
}

if (0) {
    console.log('World'); // doesn't run, 0 is falsy
}
```

---

## 5️⃣ **Default Values Using OR (`||`)**

* Use `||` to **assign a default if the user doesn’t provide input**:

```javascript
let username = prompt('Enter your name') || 'PROCODER';
let age = prompt('Enter your age') || 22;

console.log(username, age);
```

* If the user presses **Cancel** or leaves it blank:

  * `username = 'PROCODER'`
  * `age = 22`

✅ Clean, simple, and avoids undefined or null.

## 6️⃣ **Using `else` for invalid input**

You can handle invalid inputs by using `else` at the end:

```javascript
if (age < 0) {
    console.log('Please enter a valid age');
} else if (age <= 4) {
    console.log('User is a kid');
} else if (age <= 17) {
    console.log('User is a teenager');
} else if (age <= 45) {
    console.log('User is an adult');
} else if (age <= 120) {
    console.log('User is retired');
} else {
    console.log('User is immortal');
}
```

* The `else` here handles **any edge case not covered above**.
* This ensures your program is robust.

## 7️⃣ **Checking if a value exists**

You can also check if a variable exists (not null, undefined, or falsy):

```javascript
let username = prompt('Enter your name');

if (!username) { 
    username = 'PROCODER';
}

console.log(username);
```

* `!username` → true if `username` is null, undefined, empty string, or 0.
* Assigns `'PROCODER'` if the user didn’t enter a name.

## 8️⃣ **Using `if` with truthy/falsy values**

* `if` doesn’t need to compare to `true` explicitly.
* Any **truthy value** makes the `if` block run.

```javascript
if (username) {
    console.log('Hello ' + username);
} else {
    console.log('No name entered');
}
```

* Works for numbers too:

```javascript
let age = 0;

if (age) {
    console.log('Age exists');
} else {
    console.log('Age not entered'); // runs because 0 is falsy
}
```

## 9️⃣ **Best Practices from the video**

1. **Always check first conditions carefully** using `if` and `else if` to avoid unnecessary checks.
2. **Use `||` for defaults** – clean and readable.
3. **Use `else` for catch-all conditions** like invalid input.
4. **Avoid using `const`** if you need to reassign values based on input (use `let`).
5. **Keep code readable** – indent, format, and optionally use Prettier.
6. **Truthy/falsy logic** allows concise code without always comparing explicitly.

## 10️⃣ **When to use `switch` instead of `else if`**

* If you have **many conditions based on a single value**, switch-case is cleaner.

```javascript
switch (ageCategory) {
    case 'kid':
        console.log('User is a kid');
        break;
    case 'teen':
        console.log('User is a teenager');
        break;
    default:
        console.log('User is adult or retired');
}
```

* Output is the same as `if-else if-else`, but easier to read for many cases.
